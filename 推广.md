🤔 为什么做这个

我自己是 HL 重度用户,以前每次判断一个位置能不能进,流程是:

Hyperdash 看大仓 → 切 HL app 看自己持仓 → 复制地址到 Etherscan → Twitter 搜有没有人在讨论 → Telegram 翻消息 → CoinGlass 看 heatmap → 回来下单。

10 个标签页,15 分钟,只为回答一个问题。

AI agent 今年已经普及,Claude、Cursor、Codex 都支持 MCP,但它们看不懂 Hyperliquid。

我去 GitHub 把所有 HL 相关 MCP 扒了一遍,发现问题:

1. 全部都是把 HL REST API 包了一层,agent 拿到一堆 JSON,还是不知道怎么解读
2. 调用都要用户手动传地址 —— 但普通人哪知道哪些地址值得追
3. 没有人做"判断层" —— funding +12% 年化是极端还是普通?价涨 5% 是真需求还是空头挤压?


💡 三个洞察

一、Agent-native 界面会替代 dashboard
不是因为 dashboard 不好,而是因为 dashboard 需要你花时间去看。一句话直接出答案,永远赢。

二、工具的价值不是多一个端点,而是把数据变成判断
同样的 funding 数据,裸数据是 "+12%/年",有判断的工具告诉你 "funding 是过去 30 天均值的 2.5 倍,已进入极端区间,历史上这个位置出现过 3 次回调"。后者才是决策素材。

三、零配置是 UX 的底线
用户不应该自己去 Google "HL 上大佬地址",这个数据 HL 自己就公开了 —— 官方 leaderboard 34,684 行。我把它预置进去,用户装完直接用。


🧰 产品详细介绍

22 个工具分 4 大模块。


A. 清算风险(6 个)

get_top_liquidation_risks —— 扫地址列表(不传就用预置 60 个大佬),返回离爆仓最近的 N 个仓位,按距离 × 仓位大小加权

liquidation_heatmap —— 某资产在当前价 ±15% 范围内的清算密度图,多空分列

simulate_cascade —— 模拟瞬时价格冲击,3 轮迭代算连锁爆仓,返回总金额 + top 5 受损者

my_position_risk —— 自己地址输入,快速诊断哪个仓位 < 5% 要爆

simulate_my_liq_price —— 加仓 / 减仓后新爆仓价在哪,下单前试算

get_recent_liquidations —— 过去 N 小时大佬们的爆仓事件流


B. 鲸鱼与仓位流(5 个)

whale_pnl_leaderboard —— 实时拉 HL 官方榜,按日/周/月/全周期的 PnL/ROI/成交量排序

get_whale_flows —— 过去 24h 大户大单流(≥$1M)时间序列

address_position_history —— 单地址历史复盘,按资产分组,带 PnL / 胜率

smart_money_flow —— 输入一个资产,直接告诉你预置 60 位大佬整体多还是空,top 3 多头 / 空头各多少金额。零参数

get_funding_pnl —— 某地址过去 N 天付了 / 收了多少 funding,按资产拆分。套利玩家必用


C. 市场结构(8 个)

get_funding_divergence —— HL vs Binance / Bybit / OKX 同资产 funding 差

asset_snapshot —— 单资产一次性概览:价格 / funding / OI / 24h 成交

get_all_asset_ctxs —— 全 HL 200+ 合约一次性拉完,按 funding 极值 / 成交 / OI / 涨跌排序

hlp_metrics —— HLP 金库 APR、跟投人数、存款状态 + 健康度解读

orderbook_imbalance —— 盘口深度多空失衡度

compare_perps —— 任意几个资产横向比对

historical_context —— 当前 funding / 价 相比 7d / 30d 均值差多少,当前是 30d range 的百分位,realized vol 多少。把"+12%/年"翻译成"比月均高 2.5σ"

detect_anomalies —— 扫 200+ 合约,一次返回 funding 极值 top 20 + 大涨跌 top 20 + 成交异常 top 20,附 3 条 headline 总结


D. 叙事与总结(3 个,中英双语)

explain_market_structure —— 当前 HL 市场交易员视角的解读。里面有个 regime 分类器,把头条资产判为以下五种之一:

→ organic rally 自然上涨(价涨,funding 正常)
→ short squeeze 空头挤压(价涨 + funding 极高 = 多头在付溢价)
→ long squeeze 多头挤压(价跌 + funding 负 = 空头在收钱)
→ funding bleed 放血(价平但 funding 极端 = carry 单向堆积)
→ coiling 盘整

这几个标签是区分"该追还是该避"的关键,很多人亏钱就是把 short squeeze 当 organic rally 进场。

asset_snapshot_narrative —— 单资产 3 段叙事,含 squeeze 警告

daily_briefing —— 一页纸 HL 日报,中英双语
